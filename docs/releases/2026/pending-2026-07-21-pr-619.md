## 2. 范围与风险

### 变更摘要

- 加固 guardrail、repair、预约取消和副作用短路处理。
- 修复复聊停止条件、提醒调度、定位语义和模型路由。
- 报名后复聊采用“同工单明确聊天约定优先、归属不明回退工单时间”的面试时间口径。
- 修复岗位半径无结果、经验门槛单位和跨顾问预约披露口径。
- 增加 Dashboard 角色模型动态配置。
- 增加主聊、测试页、复聊反馈来源和飞书截图附件。

### 高风险区域

- **P0：生成与守卫链路**——错误放行会产生空头承诺、内部信息泄漏或错误副作用。
- **P0：复聊**——错误判定会重复打扰、漏提醒、提前发送面试后回访，或把其他工单的聊天时间串入当前预约。
- **P0：反馈附件**——请求体可能超过全局 20MB 限制，或因飞书权限导致反馈丢失。
- **P1：模型配置**——默认值和 Dashboard 覆盖变化可能造成线上回复质量漂移。
- **P1：岗位事实**——范围、经验单位和历史预约披露错误会误导候选人。

### 部署顺序与回滚

1. 先确认生产模型配置和飞书字段/权限。
2. 再滚动发布应用；无 migration 前置。
3. 发布后验证 guardrail、复聊、Dashboard 配置读取和两类反馈表。

- 回滚版本：发布前记录上一生产 tag。
- 回滚条件：P0 线上回归、复聊异常触达、模型错误率显著升高、反馈主体写入失败。
- 回滚动作：部署上一 tag；恢复原模型覆盖；必要时关闭截图入口或复聊 rollout。
