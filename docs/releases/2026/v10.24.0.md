## 2. 范围与风险

### 变更摘要

- **地理解析域（#640/#643/#646）**：建立 `resolution/geo` 唯一入口，完成消费者迁移、三轮地理信号扫描编排、海绵行政区适配器抽取、昆山市到苏州市业务映射与冲突 shadow 观测。
- **坐标与推荐纪律（#645）**：记录模型自编/无来源坐标的 provenance 与偏差，禁止凭记忆自编坐标；年龄 `hard_reject` 岗位默认不推荐。
- **拉群幂等（#647）**：企微 `errcode=-12` 或“已发送入群邀请”按邀请卡片投递成功处理，停止换群重发。
- **预约、身份与护栏（#651）**：面试具体时刻必须落在配置窗口内，线上面试使用线上成功话术；企微账号昵称/性别成为 Agent 身份锚点；“转人工”等露馅话术由 observe 升为 revise。
- **品牌解析收敛（#651）**：解析前剥离引用块；在生产归因完成后删除 legacy brand shadow 对照路径，并同步架构证据与回滚边界。

### 高风险区域

- **P1：地理查询与岗位推荐**——消费者入口迁移及昆山映射可能影响城市/区县过滤；兼容门面、golden cases 与新旧入口同引用断言锁定行为。
- **P1：面试预约**——窗口边界和线上/线下话术直接影响候选人体验；失败模式为错误拒绝合法时间或发送错误到店指引。
- **P1：身份与输出护栏**——确定性 revise 可能过度改写正常客服表述；仅命中闭集“转人工/真人经理/专人联系”等露馅模式。
- **P1：拉群**——`-12` 语义识别失败会重复发卡；回归测试锁定只调用一次并立即停止候选群遍历。
- **P2：品牌 shadow 下线**——删除的是无行为影响的观测对照组；现行品牌解析主路径与状态机保留。
