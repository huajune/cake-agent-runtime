## 2. 范围与风险

### 变更摘要

- **预约身份与取消安全（#684）**：允许候选人的明确姓名确认解除打招呼昵称负向证据；限制重复索名并转人工；要求预约手机号可追溯到候选人原文；取消工单前核对当前预约归属和真实工单状态。
- **人工接管（#684）**：明确“转人工”请求进入确定性风险通道，暂停自动回复并触发接管告警，候选人侧保持静默。
- **无岗承接与复读治理（#685）**：二次无岗时换档承接并先回答本轮问题；出站回复与历史逐字相同时进入一次受控改写。
- **类型与复聊门面收口（#686）**：退役三个无主 re-export，消费者直连真实符号位置；不改变运行时行为。
- **品牌污染拦截（#684 补充）**：将招聘平台发布 / 派遣主体“跃橙云服”从文本匹配、图片品牌 ID 契约和品类展开中剔除，保留按 ID 反查元数据能力。
- **文档整理（#684 补充）**：重写仓库入口与文档索引，将已完成或不再维护的方案移动到 `docs/archive/`，同步修正引用。

### 高风险区域

- **P0：真实预约写入**——姓名或手机号溯源误放行可能把污染记忆写入真实预约；误拒则会增加重复追问或人工介入。
- **P0：取消工单**——归属或状态判断错误可能取消错误预约；状态查询失败维持既有 best-effort 放行，需观察外部查询稳定性。
- **P0：人工接管**——漏判会继续自动回复；误判会不必要地暂停会话。关键词采用强匹配与短消息约束降低假阳。
- **P1：重复回复改写**——全等回复进入 repair；若 repair 未实际改写，现有白改保护会保留首版，需观察白改率。
- **P2：类型门面与文档归档**——理论上无行为变化，主要风险是遗漏 import 或文档链接。
