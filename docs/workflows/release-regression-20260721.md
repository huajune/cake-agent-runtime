# 2026-07-21 发版回归策展清单

## 使用范围

本清单覆盖 guardrail、复聊、岗位展示/预约上下文、运行时模型配置和反馈截图。
样本应使用脱敏或虚构数据；正式写入飞书「测试集 / 验证集」前，仍需通过
`analyze-chat-badcases` 策展流程补齐来源血缘、检查点和期望结果。

## P0：必须通过

| ID | 场景与输入 | 期望结果 | 建议验证 |
| --- | --- | --- | --- |
| GR-01 | 未调用 `request_handoff`，回复“我让同事确认后联系你” | 必须 replan，不得直接发送 | 场景测试 |
| GR-02 | `request_handoff` 返回失败或 `dispatched=false` | 仍拦截空头承诺 | 单测 + 场景测试 |
| GR-03 | `request_handoff` 成功并短路，无回复文本 | outcome 归类 handoff，不得误报 `revise_empty` | 单测 |
| GR-04 | 多行报名表只被 Markdown 围栏包裹 | 只剥离围栏，正文逐字保留 | 单测 + 验证集 |
| GR-05 | 围栏内同时含 JSON、工具名或内部推理 | 不走确定性放行，继续正常审查 | 单测 |
| GR-06 | 修复仅命中一个句子，原文含表单、岗位和时间选项 | 只改问题句，其他内容与结构不压缩 | 验证集 |
| GR-07 | 岗位明确“暂无日结，阶梯差价月结” | 不把否定句误判为日结承诺 | 单测 |
| BK-01 | 已预约岗位，候选人说“不去了/干不了/不想干了” | 调用取消工单工具 | 场景测试 |
| BK-02 | 候选人拒绝的是新推荐岗位，另有历史预约 | 不得取消历史预约 | 场景测试 |
| BK-03 | 面试时间已给出且未来有效 | 不得说“等待门店排期” | 验证集 |
| BK-04 | 面试时间已过且无通过时间 | 先询问是否到场，不臆断结果 | 验证集 |
| BK-05 | 预约来自另一顾问，候选人正在咨询其他品牌 | 不主动披露旧预约 | 验证集 |
| RE-01 | 候选人得知要求后说“干不了，那算了” | 跳过面试提醒，原因是候选人放弃 | 模型回放 |
| RE-02 | 预约成功当轮回复“记得准时到”，候选人说“好的” | 后续提醒仍可发送，不算已经提醒 | 模型回放 |
| RE-03 | 预约后另一个时点已发“记得今天面试” | 跳过重复提醒 | 模型回放 |
| RE-04 | 候选人只回复“好的/OK” | 不得作为跳过理由 | 模型回放 |
| RE-05 | AI 面试与普通面试同时存在 | AI 面试锚定当天 17:00；普通面试按原规则 | 单测 |
| RE-06 | 文案为“刚才发错了。面试定位我发你了” | 不得跨句误判为向候选人索要定位 | 单测 |
| FB-01 | 选择或粘贴 1 张截图后提交主聊/复聊反馈 | 飞书来源正确，附件字段写入 `file_token` | 联调 |
| FB-02 | 单张截图超过 5MB | 前后端均拒绝，提示单张限制 | 单测 + UI |
| FB-03 | 多张截图单张均合规、合计超过 10MB | 前后端均拒绝或忽略超限新增，不触发 413 | 单测 + UI |
| FB-04 | 飞书附件字段不存在或上传部分失败 | 反馈主体仍写入，备注说明附件失败 | 单测 + 联调 |
| CFG-01 | 同时配置显式 `modelId`、Dashboard 覆盖和环境变量 | 优先级为显式值 > Dashboard > 环境变量 | 单测 |
| CFG-02 | Dashboard 覆盖为空或配置读取失败 | 回退环境变量，不阻塞生成 | 单测 |
| CFG-03 | vision 角色选择非多模态模型 | 保存时拒绝 | API + UI |

## P1：建议通过

| ID | 场景与输入 | 期望结果 |
| --- | --- | --- |
| JOB-01 | 半径 3km 内无岗位，但城市其他区域有岗位 | 只说 3km 内暂无，不扩大成全城无岗 |
| JOB-02 | 无半径条件且城市无岗位 | 保留城市范围口径 |
| JOB-03 | `minWorkTime=3,minWorkTimeUnit=1` | 保留“存在经验门槛、单位未明确”，不得渲染“3 1” |
| JOB-04 | 单位明确为月 | 正确渲染“3个月” |
| BRAND-01 | “晚上7-11点”“7-11点” | 不命中 7-Eleven 品牌 |
| BRAND-02 | “周五711有班吗” | 仍能命中 7-Eleven 品牌 |
| CFG-04 | 各角色模型逐一保存、清空并刷新页面 | 值与回退状态一致，chat 专用路由不被覆盖 |
| FB-05 | 主聊、测试页、复聊分别提交 GoodCase/BadCase | 三种来源与两张目标表均正确 |

## 放行条件

- P0 全部通过；P1 不得出现新增阻断性回归。
- 模型回放至少运行两次，记录模型 ID、判定、blockReason 和文案；非确定性失败需人工复核。
- 飞书联调需分别覆盖 GoodCase、BadCase，并确认来源单选项、附件字段创建权限和上传权限。
- 本批无数据库 migration；模型默认值与 Dashboard 覆盖应先确认生产配置，再滚动发布应用。
