# Cake Agent Runtime 架构知识库（MOC）

> 项目一句话：企微渠道的招聘 AI Agent 运行时——候选人在微信里和"招聘顾问"对话，背后是 NestJS + 多模型 LLM + 四层记忆 + 出站守卫的生产系统，服务餐饮连锁企业蓝领招聘，含运营 Dashboard（React）。
> 本知识库只收录**复杂且经过真实权衡/踩坑调整**的机制，每篇都带"学习要点"段。

## 一条消息的旅程（把所有笔记串起来的主线）

```
候选人企微消息 → 回调入口(200 快速返回)
 → [[02-消息debounce聚合与租约锁]]（静默窗口聚合 + 90s 租约锁）
 → [[03-四层记忆系统]] 召回 + [[05-Prompt-Section动态组装体系]] 拼 prompt
 → [[01-多模型三层容错架构]] 选模型执行，[[12-工具层设计与幻觉防御]] 多步工具调用
 → [[07-出站守卫三档裁决链]] 审查（误拦治理 [[08-守卫假阳治理与业务裁定]]）
 → [[06-Agent回合编排与TurnFinalizer]] 修复 + 副作用结算
 → [[11-拟人化投递]] 分段发送
 → 全程 [[10-可观测性体系]] 落档案；事实写回见 [[04-Facts并发安全与字段级写入]]
候选人消失后 → [[09-复聊独立生成链路]] 主动触达
```
