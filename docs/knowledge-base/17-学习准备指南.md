# 学习准备指南（8 年前端 → AI 应用全栈）

## 一、定位：你卖的是什么

不要把自己定位成"转后端的前端"（那是降维竞争，跟科班后端拼数据库调优拼不过）。你的定位是 **AI 应用工程师 / LLM 全栈**——2026 年最稀缺的画像恰恰是：**8 年工程功底 + 独立交付过生产级 Agent 系统 + 前后端都能动手**。市面上大量"会调 API 的 prompt 工程师"和大量"没碰过 LLM 的传统后端"，两边都有的人极少。

## 二、关于"AI 辅助开发"怎么说（最敏感的问题）

**主动说，别藏。** 藏着必然在深挖时露馅；说好了反而是加分项。推荐口径：

> "这个项目是我一个人借助 AI 编程工具完成的。我的角色更像架构师 + 技术负责人：**架构分层、并发语义、事故根因、每一个技术裁定都是我做的**，AI 负责加速编码。生产系统的难点从来不在写代码，在于知道该写什么——debounce 还是固定窗口、锁租约设多长、守卫误拦怎么灰度，这些判断 AI 给不了。"

然后立刻用细节证明（深入学习追问什么你都接得住）：讲 [[02-消息debounce聚合与租约锁]] 的三时长约束，讲 [[14-生产事故复盘集]] 的 522 排查。**能讲清 why 的人不怕被质疑 who 写的 how。**

诚实的边界：如果被问"哪段代码你手写的"，答"具体实现大多是 AI 生成、我 review 和返工的，但我可以现场白板推导任何一个模块的设计"——把重点引向你的强项。

## 三、三个必备叙事（提前排练到 5 分钟脱稿）

1. **系统全景（5 分钟版）**：照 [[00-MOC-架构知识库总览]] 的"一条消息的旅程"讲，从企微回调到拟人化投递。练到能在白板上边画边讲。
2. **最难的技术问题**：首推**发版丢消息事故**（[[14-生产事故复盘集]] 案例 2）——涉及分布式锁、TTL、优雅停机、静默失败，层次最丰富；备选 facts 并发丢更新（[[04-Facts并发安全与字段级写入]]）。
3. **一个体现判断力的裁定**：首推复聊不复用主 generator（[[09-复聊独立生成链路]]）——能讲出"主动触达风险剖面不同所以生成策略更保守"，是架构判断力的直接证据。

## 四、高频问题预演

| 问题 | 弹药 |
|---|---|
| Agent 幻觉怎么治？ | 纵深防御：入口闸门→工具入参确定性兜底→出口守卫比对工具真值→provenance 校验（[[12]]、[[08]] 案例4） |
| LLM 应用怎么测试？ | 确定性单测 + badcase 回归集 + shadow 影子运行 + 生产抽样闭环（[[13-测试与评估体系]]） |
| 消息乱序/重复怎么处理？ | 去重 + debounce + 租约锁 + 幂等（[[02]]） |
| 上下文窗口怎么管理？ | 四层记忆按生命周期分层 + 读时三重裁剪 + 空闲沉淀（[[03]]） |
| 模型挂了怎么办？ | 错误三分类 + 指数退避 + fallback 链 + 角色路由热切换（[[01]]） |
| 印象最深的 bug？ | 六选一，按"现象→排查→根因→止血/根治→防复发"结构讲（[[14]]） |
| 为什么不用 LangChain/RAG？ | 结构化事实用 schema 字段比 embedding 可靠可审计；框架抽象不匹配业务管道，Vercel AI SDK 薄封装够用 |
| 成本怎么控？ | 角色路由分级用模型、复聊模板档零 LLM、守卫 LLM 档只在高风险触发、debounce 合并回合 |

## 五、补短板清单（学习前突击优先级）

1. **高**：Postgres 基础再夯一层——索引原理（B-tree、为什么非 unique 索引撑不起 ON CONFLICT）、事务隔离级别、EXPLAIN 看执行计划。你有 TOAST/连接池的实战故事，但要能接住基础追问。
2. **高**：Redis 数据结构与过期机制（hash/list/TTL/惰性删除），你的锁和 factsv2 故事都建立在上面。
3. **中**：Node.js 事件循环 + AsyncLocalStorage 原理（有 [[10]] 的实战，补理论即可）。
4. **中**：消息队列通识（至少能对比 Bull/Redis 队列 vs Kafka 的适用场景，承认选型是"够用优先"）。
5. **低**：K8s/微服务大厂八股——你的项目是单体+队列，别硬凹，大方说"这个规模单体是正确选型，过早微服务是负债"。

## 六、利用好前端背景

- Dashboard 是你的主场：React 18 + Vite、KPI 看板、消息处理页性能优化（52s→亚秒，前端感知的问题、数据库层的修复——完美的全栈故事）
- Prompt Section 体系 = 组件化思想迁移到 prompt 工程（[[05]]），这个类比只有前端出身讲得出
- 拟人化投递 = 产品体验思维进入后端架构（[[11]]）

## 七、最后一条

学习前一晚只看 [[00-MOC-架构知识库总览]] 和本文，不要试图背所有笔记。你不是在背别人的项目——这些坑是你自己踩的，事故是你自己修的，你只需要把它们**讲成故事**。
